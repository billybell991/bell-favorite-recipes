Adding a New Recipe

  1. Go to the content/recipes/ folder
  2. Copy any existing .md file and rename it (use lowercase, hyphens instead of spaces, e.g. grandmas-apple-pie.md)
  3. Open the new file in a text editor (VS Code, Notepad, etc.)
  4. Change the content between the --- lines (the "front matter") and the recipe text below
  5. Save the file

Recipe File Format

---
title: "Grandma's Apple Pie"
categories: ["Family Recipes"]
image: "https://images.pexels.com/photos/1126359/pexels-photo-1126359.jpeg?auto=compress&cs=tinysrgb&w=600"
---

## Ingredients

- 6 cups sliced apples
- 3/4 cup sugar
- 2 tbsp flour
- 1 tsp cinnamon

## Directions

1. Preheat oven to 425°F
2. Mix apples with sugar, flour, and cinnamon
3. Pour into pie crust
4. Bake for 40-50 minutes

Deleting a Recipe

  1. Delete the .md file from content/recipes/

Preview Your Changes Locally

Before publishing, you can see how the site looks on your computer:

  1. Open a terminal (PowerShell) in VS Code or Windows
  2. Navigate to your project folder:
cd "c:\Stuff\Bell_Recipes_Project"
  1. Run the Hugo dev server:
hugo server --baseURL http://localhost:1313/
  1. Open your browser to: http://localhost:1313/bell-favorite-recipes/
  2. Press Ctrl+C in the terminal when you're done previewing

Publishing Changes to the Web

After making changes and you're happy with how they look:

  1. Open a terminal (PowerShell) in your project folder
  2. Run these two commands:

Step 1: Build the site

hugo --gc --minify --baseURL "https://billybell991.github.io/bell-favorite-recipes/"

Step 2: Deploy to GitHub

powershell -NoProfile -ExecutionPolicy Bypass -File deploy.ps1
Note: After deploying, it takes 1-2 minutes for GitHub to update the live site. Just refresh the page after a couple minutes.

Troubleshooting

"hugo" is not recognized

Run this first to refresh your PATH, then try again:

$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")

Port 1313 already in use

Another Hugo server is still running. Stop it first:

Get-Process hugo -ErrorAction SilentlyContinue | Stop-Process -Force

Site looks unstyled locally

Make sure you use the --baseURL flag when running locally:

hugo server --baseURL http://localhost:1313/

Deploy script fails

Make sure you have internet access and are logged into GitHub. If git asks for credentials, enter your GitHub username and personal access token (not your password).
